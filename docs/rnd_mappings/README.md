# R&D Mappings: AS-IS to ERPNext

This directory contains detailed mappings for each of the 5 R&D processes from the AS-IS_Process.xlsx R&D sheet.

Each process has its own .md file with:
- Exact AS-IS steps from the sheet
- Broken down ERPNext implementation steps
- Pain points identified
- How ERPNext solves them

## Process Files

| S.No | Process Title | File |
|------|---------------|------|
| 1 | Customer Order execution | [customer_order_execution.md](customer_order_execution.md) |
| 2 | Approach with Procurement Team | [approach_procurement.md](approach_procurement.md) |
| 3 | Handover Process | [handover_process.md](handover_process.md) |
| 4 | Repair & Replacement | [repair_replacement.md](repair_replacement.md) |
| 5 | Sample Dispatch for Customers | [sample_dispatch_customers.md](sample_dispatch_customers.md) |

## Process Summaries

### 1. Customer Order execution
**AS-IS Overview**: Manual SOR, documents not centralized.
**ERPNext Usage**: Projects and Tasks.
**Key Improvements**: Centralized tracking, collaboration.

### 2. Approach Procurement
**AS-IS Overview**: Verbal communication, no approvals.
**ERPNext Usage**: Purchase Requisition.
**Key Improvements**: Formal requests, approvals.

### 3. Handover Process
**AS-IS Overview**: Email-based handover.
**ERPNext Usage**: Project milestones.
**Key Improvements**: Structured handovers.

### 4. Repair & Replacement
**AS-IS Overview**: Physical handovers, no tracking.
**ERPNext Usage**: Delivery Notes for returns.
**Key Improvements**: Traceable processes.

### 5. Sample Dispatch
**AS-IS Overview**: Manual dispatch based on type.
**ERPNext Usage**: Delivery Note with custom fields.
**Key Improvements**: Automated routing, tracking.
